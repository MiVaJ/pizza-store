# Modern Pizza Store (Учебный проект)

⚠️ **ВНИМАНИЕ**: Данный сайт является учебным проектом. Интерфейс симулирует реальный коммерческий продукт, однако все заказы, оплаты и пользователи являются демонстрационными. Пицца не доставляется в реальности.

## Стек технологий

- **Backend**: FastAPI, SQLAlchemy 2.0 (Async), Asyncpg, Alembic, PostgreSQL (в Docker)
- **Task Queue**: Celery, Celery Beat, Redis (брокер и кэш)
- **Frontend**: React (Vite), Tailwind CSS, Zustand, Axios
- **Инфраструктура**: Docker, Docker Compose, GitHub Actions (CI)
- **Мониторинг**: Sentry (логирование ошибок), Яндекс Метрика (аналитика целей)

## Переменные окружения

Перед первым запуском скопируйте файл с примером и заполните значения:

```bash
cd backend
cp .env.example .env
```

_Обязательно укажите `SECRET_KEY` - без него авторизация не будет работать._

## Запуск через Docker (рекомендуется)

Самый простой способ поднять весь проект одной командой.

### 1. Запуск всех сервисов

Из корневой папки проекта:

```bash
docker compose up --build
```

Это поднимет все контейнеры:

- **postgres** - база данных на порту `5433`
- **redis** - брокер задач и кэш на порту `6379`
- **celery_worker** - воркер фоновых задач
- **celery_beat** - планировщик задач
- **backend** - FastAPI на порту `8000`
- **frontend** - React-приложение на порту `5173`

### 2. Применение миграций и заполнение меню

После первого запуска (в новом терминале) для загрузки миграции и сидинга:

```bash
docker compose exec backend poetry run alembic upgrade head
docker compose exec backend poetry run python -m src.seeds.seed_pizzas
```

### 3. Готово

- Сайт: http://localhost:5173
- API / Swagger: http://localhost:8000/docs

### Последующие запуски

Если код не менялся, миграции и сиды уже применены:

```bash
docker compose up
```

Если менялся `Dockerfile` или зависимости - пересоберите образы:

```bash
docker compose up --build
```

## Запуск без Docker (для разработки)

Проект разработан в виде монорепозитория. Бэкенд и фронтенд запускаются раздельно, а база данных работает в изолированном Docker-контейнере.

### 1. Запуск Базы Данных (Docker)

Находясь в корневой папке проекта, убедитесь, что запущен Docker Desktop, и выполните:

```bash
docker compose up -d
```

_Это поднимет PostgreSQL 16 на порту 5433._

### 2. Запуск Бэкенда (FastAPI)

Перейдите в папку бэкенда, установите зависимости через Poetry и запустите сервер:

```bash
cd backend

# Создание виртуального окружения (ОБЯЗАТЕЛЬНО):
python -m venv .venv

# Активация окружения:
.venv\Scripts\activate      # Для Windows (PowerShell)
source .venv/bin/activate   # Для Linux/macOS

# Установка всех зависимостей проекта:
poetry install

# Запуск сервера:
poetry run uvicorn src.main:app --reload
```

_Интерфейс API будет доступен по адресу: http://127.0.0.1:8000_

_Интерактивная документация (Swagger): http://127.0.0_

### 3. Управление базой данных и миграциями (Alembic)

После успешного запуска бэкенда необходимо применить существующие миграции к базе данных в Docker и наполнить меню стартовыми позициями.

#### Применение миграций

```bash
cd backend
poetry run alembic upgrade head
```

#### Первоначальное наполнение меню пиццами (Seeding)

Скрипт добавит в базу данных 5 стартовых пицц для проверки адаптивной сетки фронтенда:

```bash
cd backend
poetry run python -m src.seeds.seed_pizzas
```

### 4. Запуск Фронтенда (React + Vite)

#### Первый запуск (Установка)

Выполняется один раз для скачивания всех библиотек из npm:

```bash
cd frontend
npm install
```

#### Последующие запуски

```bash
cd frontend
npm run dev
```

_Сайт откроется по адресу: http://localhost:5173_

## Роли пользователей

В системе предусмотрено 4 роли с разным уровнем доступа:

| Роль        | Возможности                                                                    |
| ----------- | ------------------------------------------------------------------------------ |
| **client**  | Просмотр меню, оформление заказов, просмотр своей истории заказов и статистики |
| **manager** | Все возможности клиента + просмотр всех заказов и изменение их статуса         |
| **courier** | Роль зарезервирована в системе, функционал пока не реализован                  |
| **admin**   | Полный доступ ко всем функциям системы                                         |

### Регистрация персонала (manager, admin)

На текущем этапе форма регистрации на сайте создаёт пользователей только с ролью `client`. Для создания персонала:

1. Зарегистрируйте пользователя через форму на сайте или `POST /api/auth/register` в Swagger (http://127.0.0.1:8000/docs)
2. Измените поле `role` напрямую в базе данных (через любой DB-клиент, например DBeaver или psql)

_Эндпоинт для смены роли через API пока не реализован - это планируется добавить в будущих версиях._

## Фоновые задачи (Celery)

Проект использует Celery Beat для автоматического выполнения задач по расписанию.

### Автоматическая отмена заказов

Заказы со статусом `pending` автоматически отменяются через 30 минут после создания.
Задача запускается каждую минуту через Celery Beat, брокер - Redis.

Для локального запуска воркера и планировщика вне Docker:

```bash
cd backend

# Воркер
poetry run celery -A src.worker.celery_app worker --loglevel=info

# Планировщик (в отдельном терминале)
poetry run celery -A src.worker.celery_app beat --loglevel=info
```

## Качество кода и Линтеры (Бэкенд)

Проект использует современные инструменты автоматической проверки кода:

- **Ruff**: Быстрый линтер и форматировщик кода.
- **MyPy**: Инструмент строгой статической проверки типов Python.

### Запуск проверок вручную:

```bash
cd backend
poetry run ruff check .          # Проверка логических ошибок и импортов
poetry run ruff check . --fix    # Автоматическое исправление логических ошибок и импортов
poetry run ruff format --check . # Проверка стиля и отступо
poetry run ruff format .         # Автоматическое исправление стиля кода
poetry run mypy .                # Проверка строгой типизации
```

## Тестирование

Проект покрыт тестами на pytest, включая проверку эндпоинтов авторизации и заказов.

```bash
cd backend
poetry run pytest
```

### CI

При push и pull request в `main`, затрагивающих папку `backend/`, автоматически запускаются линтеры (Ruff), проверка типов (MyPy) и тесты (pytest) через GitHub Actions.
